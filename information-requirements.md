# Information and Requirements

เอกสารนี้กำหนดกรอบเนื้อหาสำหรับบทความ/บทเรียนหัวข้อ  
"IoT Immersion: Experience the Professional AIoT Platform"

## Scope (Current Phase)

ในเฟสนี้จะเผยแพร่เป็นตอนสั้นต่อเนื่อง **M1–M6** เพื่อให้ครอบคลุมทั้งแนวคิด การตั้งค่าระบบ การใช้ทรัพยากรฟรี การใช้ทรัพยากรที่มีเงื่อนไขสิทธิ์ และการคัดเลือกโมเดลเพื่อใช้งานจริง

- **M1 - Introduction to IoT Immersive and Digital Twin**
- **M2 - Introduction to TESAIoT Digital Twin Platform**
- **M3 - Installation, Connectivity, and Setup Validation**
- **M4 - Free Loader: Free GitHub Asset Pack:** ซิงค์ทรัพยากร 3D/เทกซ์เจอร์ฟรีจากที่เก็บ GitHub ผ่าน Free Loader; แยกความหมายจาก Model Loader; รองรับทั้ง VS Code Extension และ Web Browser (พร้อมหมายเหตุ bridge เมื่อพัฒนาบนเว็บ)
- **M5 - Model Loader: Permission-based Asset Access** การอนุญาตเข้าถึงแหล่งทรัพยากร การตั้งค่า Base URL/API Key/CA cert และการใช้งานเมื่อได้รับสิทธิ์
- **M6 - Model Catalog: Organize and Preview Models** รวมโมเดลจากหลายแหล่ง ค้นหา/กรองหมวดหมู่ พรีวิว 3D และคัดเลือกโมเดลก่อนนำไปใช้จริง

## Direction

- **Education Tone:** ใช้ภาษาทางการในแนวบทเรียน/อบรมวิชาชีพ (ชัดเจน เป็นระเบียบ ให้เกียรติผู้อ่าน) แต่**ไม่แข็งเกินจนเข้าถึงยาก** ประโยคควรสั้นพอประมาณ แบ่งย่อหน้าให้อ่านสบายตา
- ผู้อ่านหลัก **ไม่ใช่โปรแกรมเมอร์** เขียนให้นักพัฒนาผลิตภัณฑ์ นักธุรกิจ นักพัฒนาโปรดักต์ นักการตลาด ครูและอาจารย์ นักศึกษา และผู้พัฒนาระบบสมองกลฝังตัวอ่านเข้าใจได้; นำเสนอเป็นซอฟต์แวร์สำเร็จรูป **ถ้าจำเป็นต้องใช้ศัพท์เทคนิค ให้มีคำอธิบายสั้นในวงเล็บหรือประโยคถัดไป** และเชื่อมกับประโยชน์ในงานจริงเสมอ
- เลี่ยงเครื่องหมาย em dash (—) ในประโยคภาษาไทย ใช้คำเชื่อม จุด วรรค หรือวงเล็บแทน
- ใช้ภาษาที่อ่านง่าย น่าใช้ และดึงดูดความสนใจ โดยเน้นว่าเครื่องมือช่วยงานจริงอย่างไร (ประหยัดเวลา ลดความเสี่ยง นำเสนอได้มั่นใจ ฯลฯ)
- เน้น Business Value และความมั่นใจในการนำไปต่อยอดเชิงธุรกิจ
- ไม่เน้นการเขียนโค้ด (no coding)
- ใช้งานได้ทั้งผ่าน VS Code Extension และ Web Browser (รวมการเข้าถึงผ่านโทรศัพท์มือถือและแท็บเล็ต)
- สื่อสารข้อมูลได้ทั้งกับฮาร์ดแวร์จริงและ Cloud Server ผ่าน IoT Protocol
- การเชื่อมต่อคลาวด์ในบทติดตั้งต้องครอบคลุมแนวทาง secure communication (เช่น MQTT over TLS/WSS และ credential setup)
- รองรับการทำงานร่วมกับ MCP และ LLM เพื่อสั่งการระบบด้วยภาษาคน (conversational command) ในรูปแบบคล้าย Chatbot
- workshop ใช้บอร์ด **PSoC Edge AI** ที่มี firmware พร้อมใช้งาน

## Target learners

- นักพัฒนาผลิตภัณฑ์ นักพัฒนาโปรดักต์ นักธุรกิจ นักการตลาด
- ครู อาจารย์ นักเรียน นักศึกษา และผู้สนใจเรียนรู้ Digital Twin ในเชิงประยุกต์
- ผู้พัฒนาระบบสมองกลฝังตัว (งานฝังตัวและ IoT โดยไม่จำเป็นต้องเน้นผู้เขียนโปรแกรมทั่วไปเป็นหลัก)
- (เสริมตามบริบท) วิศวกรโรงงานและผู้ใช้งานมืออาชีพที่ต้องการเครื่องมือสำเร็จรูป

## Publication goal for this phase

สร้างชุดเนื้อหาที่อ่านจบเร็ว เข้าใจภาพรวมไว และเห็นแนวทางนำเครื่องมือไปใช้จริงในงานและธุรกิจได้ทันที ครอบคลุมตั้งแต่ M1–M6 เพื่อให้ผู้เรียนใช้งานแพลตฟอร์มได้ครบทั้งเส้นทางฟรี เส้นทางที่ต้องมีสิทธิ์เข้าถึง และการคัดเลือกโมเดลก่อนใช้งานจริง

## Content flow guideline

เพื่อให้เนื้อหาอ่านลื่นและไม่สะเปะสะปะ ให้ใช้ลำดับการเล่าเรื่องเหมือนกันทุกตอน โดยคง **Education Tone** แต่คิดถึงผู้อ่านที่ไม่ใช่โปรแกรมเมอร์เป็นหลัก:

1. **Context and Value:** เริ่มจากภาพรวมและคุณค่าที่ผู้เรียน/ผู้ใช้งานจะได้รับ
2. **Core Concept and Capability:** อธิบายแกนแนวคิดและความสามารถหลักของแพลตฟอร์ม
3. **Practical Example:** ยกตัวอย่างการใช้งานที่เห็นภาพและเชื่อมกับกลุ่มเป้าหมาย
4. **Business and Learning Impact:** ปิดท้ายด้วยผลลัพธ์เชิงงานและเชิงธุรกิจที่ชัดเจน
5. **Bridge to Next Module:** มีประโยคส่งต่อไปยังบทถัดไปทุกครั้ง
